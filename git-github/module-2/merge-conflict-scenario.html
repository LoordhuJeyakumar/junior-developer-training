Understanding Git Merge Conflicts with Real-World Examples

The Recipe Book Analogy

Imagine you're working on a collaborative recipe book:

Original Recipe

Chocolate Cake:
- 2 cups flour
- 1 cup sugar
- 1/2 cup cocoa
- 2 eggs
- 1 cup milk

Chef Alice's Changes

Chocolate Cake:
- 2 cups flour
- 1 cup sugar
- 1/2 cup cocoa
- 3 eggs
- 1 cup milk
- 1/2 cup butter

Chef Bob's Changes

Chocolate Cake:
- 2 cups flour
- 1 cup sugar
- 1/2 cup dark cocoa
- 2 eggs
- 1 cup milk
- 1 shot espresso

Merge Conflict!

Chocolate Cake:
- 2 cups flour
- 1 cup sugar
<<<<<<< HEAD
- 1/2 cup cocoa
- 3 eggs
- 1 cup milk
- 1/2 cup butter
=======
- 1/2 cup dark cocoa
- 2 eggs
- 1 cup milk
- 1 shot espresso
>>>>>>> feature-bob

Resolved Recipe

Chocolate Cake:
- 2 cups flour
- 1 cup sugar
- 1/2 cup dark cocoa
- 3 eggs
- 1 cup milk
- 1/2 cup butter
- 1 shot espresso

Programming Example: Merge Conflict in Code

Original Code

function greet(name) {
  // Basic greeting
  return "Hello, " + name + "!";
}

Developer A's Changes

function greet(name) {
  // Modern JS syntax
  return `Hello, ${name}!`;
}

Developer B's Changes

function greet(name) {
  // Extra enthusiasm
  return "Hello, " + name + "!!";
}

Merge Conflict in Code

function greet(name) {
<<<<<<< HEAD
  // Modern JS syntax
  return `Hello, ${name}!`;
=======
  // Extra enthusiasm
  return "Hello, " + name + "!!";
>>>>>>> feature-enthusiasm
}

Resolved Code

function greet(name) {
  // Modern JS syntax with enthusiasm
  return `Hello, ${name}!!`;
}

Steps to Resolve Merge Conflicts

  1. Identify the conflict - Use git status to see conflicted files
  2. Open the file - Look for conflict markers (<<<<<<<, =======, >>>>>>>)
  3. Edit the file - Decide which changes to keep or combine them
  4. Remove conflict markers - Delete <<<<<<<, =======, and >>>>>>> lines
  5. Stage the changes - git add <filename>
  6. Complete the merge - git commit
